Data Manipulation Language (DML)

Data Manipulation Language (DML) is a subset of SQL (Structured Query Language) used to manage and manipulate data within database tables. DML commands are essential for retrieving, inserting, updating, and deleting data in a database. Unlike DDL (Data Definition Language), which focuses on the schema or structure of the database, DML deals with the data itself.

Key DML Commands

SELECT:

Used to retrieve data from one or more tables.

Example:

Selecting all columns from the "employees" table:

SELECT * FROM employees;

INSERT:

Used to add new rows of data to a table.

Example:

Inserting a new employee into the "employees" table:

INSERT INTO employees (employee_id, first_name, last_name, hire_date) VALUES (1, 'John', 'Doe', '2023-01-01');

UPDATE:

Used to modify existing data within a table.

Example:

Updating the last name of an employee with ID 1:

UPDATE employees SET last_name = 'Smith' WHERE employee_id = 1;

DELETE:

Used to remove rows of data from a table.

Example:

Deleting an employee with ID 1 from the "employees" table:

DELETE FROM employees WHERE employee_id = 1;

Characteristics of DML Commands

Transactional:

DML commands are transactional, meaning changes can be committed or rolled back to ensure data integrity.

Data Manipulation:

These commands are fundamental for manipulating data within the database, allowing for the insertion, update, and deletion of data as needed.

Query Capabilities:

DML provides powerful query capabilities to filter, sort, and retrieve specific data from the database.
